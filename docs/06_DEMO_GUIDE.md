# OmniMind OS - Demo Guide

This guide explains how to run the OmniMind OS CLI demo and interpret its output.

## Running the Demo

Ensure you have Python 3.9+ installed and run:

```bash
python -m src.main
```

## Understanding the Output

The CLI output tells a structured story of how OmniMind OS processes tasks.

### 1. 📥 USER INPUT
The raw prompt provided by the user (e.g., "research solar cold storage in Bangladesh").

### 2. 🎯 EXECUTION PLAN
The Planner decomposes the input into abstract capabilities. In the MVP, it generates a single step requiring the `research` capability.

### 3. 🔍 AGENT DISCOVERY
The `AgentRegistry` and `AgentCatalog` search for available agents that provide the `research` capability. You will see a table of candidates showing their source (Local, GitHub, HuggingFace), cost, and score.

### 4. ⚡ AGENT SELECTION
The `AgentSelector` applies `UserPreferences` (e.g., `prefer_free=True`, `prefer_local=True`) to pick the optimal agent from the candidate list.

### 5. 📊 EXECUTION RESULTS
The output generated by the selected agent or tool.

### 6. 📋 EXECUTION TIMELINE
An auditable trace of every step taken, verifying which agent executed which capability and its final status. This is the core explainability feature of OmniMind OS.

## Customizing the Demo

To test different inputs, you can modify the `user_input` string in `src/main.py`:

```python
if __name__ == "__main__":
    user_input = "your custom prompt here"
```
